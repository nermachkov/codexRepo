# Roadmap and Backlog

## Phase 0: Documentation

Status: in progress.

Deliverables:

- Project brief.
- Product requirements.
- Game design structure.
- Technical architecture.
- Google Play release plan.
- Privacy and compliance plan.
- Monetization plan.
- Asset pipeline.
- QA plan.
- Open questions.

## Phase 1: Concept Lock

Deliverables:

- Confirm color by number product direction.
- Working title.
- Core mechanic.
- Visual direction.
- MVP scope.
- First release platform path.
- Initial artwork categories.
- MVP artwork count.
- Content production workflow.
- Three SVG prototype artwork concepts.

Exit criteria:

- We can explain the coloring app in one sentence.
- We can describe the first 60 seconds.
- We know what the player does repeatedly.
- We know what makes the player come back.
- We know how artworks become playable numbered pages.

## Phase 2: Technical Prototype

Deliverables:

- Project scaffold.
- Basic scene.
- Input prototype.
- Rendering prototype.
- Save/settings prototype.
- Performance baseline.
- One playable coloring page.
- Zoom/pan prototype.
- Region fill prototype.
- Progress save prototype.

Exit criteria:

- Runs locally.
- Works on Android browser.
- Input feels plausible.
- No major performance red flags.

## Phase 3: Vertical Slice

Deliverables:

- Main menu.
- One polished coloring mode.
- 10-20 starter artworks.
- Result screen.
- Settings.
- Audio.
- Basic progression.
- Completed gallery.
- Hint system.

Exit criteria:

- A tester can play without explanation.
- Session has a beginning, middle, and end.
- Visual style is coherent.

## Phase 4: Android Packaging

Deliverables:

- Packaging path selected.
- Android debug build.
- Android release build.
- App icon.
- Splash behavior.
- Back button behavior.
- App lifecycle handling.

Exit criteria:

- Installable Android build works on device.
- Release build can be uploaded to internal testing.

## Phase 5: Store Preparation

Deliverables:

- Store listing.
- Screenshots.
- Privacy policy.
- Data Safety form draft.
- Content rating.
- Closed testing plan.

Exit criteria:

- Internal testing upload succeeds.
- No known P0 or P1 issues.

## Phase 6: Public Release

Deliverables:

- Production release candidate.
- Rollout plan.
- Support plan.
- Post-launch bug triage process.
